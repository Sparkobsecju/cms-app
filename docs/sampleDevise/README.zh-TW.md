# sampleDevise — 警世程式碼範例 (學一次，別再犯)

這個資料夾是**收錄本程式碼庫中真實錯誤及其修正方式的教學檔案庫。**
每份檔案聚焦於一個危險的寫法，並排展示*修正前*與*修正後*的程式碼，而且——最重要的是——明確說明**這個危險真正會咬人的具體情境**，好讓下一個人在動手打出這段程式碼之前，就先認出這個錯誤的樣貌。

這些並不是抽象的「最佳實務」。這裡的每一個範例，都是 `CMS.API` 中一個真實存在的缺陷，由安全性審查 (`/cso`) 或程式碼審查所抓出。請閱讀情境，而不只是看 diff：重點在於記住*什麼時候*這段看似安全的程式碼其實是錯的。

## 如何閱讀一個範例

1. **有漏洞的程式碼** — 出錯的確切幾行程式。
2. **為何它看起來沒問題** — 一位稱職的開發者為何仍會這樣寫的原因。
3. **攻擊情境** — 一步一步說明攻擊者是誰、他們做了什麼。這是需要牢記的部分。
4. **修正方式** — 修正後的程式碼。
5. **原則** — 一句「別做 X，改做 Y」的結論，讓你不必重讀整份檔案就能套用。
